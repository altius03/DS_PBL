# Project Overview

## 프로젝트 스냅샷
- 주제: 영화
- 분석 주체: 배급사
- 데이터 범위: 한국 일반영화
- 분석 기간: 2016~2025
- 시기 구간: 2016~2019 / 2020~2022 / 2023~2025
- 데이터 소스: KOBIS 우선 사용
- 현재 단계: 1차 과제용 데이터 확보 및 분석 준비 완료

## 프로젝트 방향
- 1차 과제는 EDA 중심으로 진행한다.
- 2차 과제는 종속변수와 피처의 관계를 분석해 적합한 피처 2개 이상을 선택하는 방향으로 확장한다.
- 1차에서는 설명 가능한 그래프와 해석 문장을 우선 확보한다.

## 분석 주체
### 배급사
- 배급사는 장르, 개봉 시기, 관람등급, 스크린 수, 상영 횟수와 흥행 성과를 함께 고려하는 주체다.
- KOBIS 데이터 구조와도 가장 잘 맞아서 1차 EDA와 2차 확장 모두에 유리하다.

## 분석 목적 5개
1. 배급사 입장에서 어떤 장르의 영화가 더 높은 흥행 성과를 보이는가
2. 배급사 입장에서 어떤 개봉 시기가 흥행에 유리한가
3. 배급사 입장에서 관람등급에 따라 흥행 성과 차이가 존재하는가
4. 배급사 입장에서 스크린 수와 상영 횟수는 흥행 성과와 어떤 관계가 있는가
5. 배급사 입장에서 코로나 이전·코로나기·최근 구간에 흥행 패턴은 어떻게 달라졌는가

## 데이터 범위 결정
### 포함
- 한국 영화
- 일반영화
- 2016~2025 개봉작

### 제외
- 외국 영화
- 독립영화
- 예술영화

### 선택 이유
- 비교 집단이 더 균질해서 해석이 선명하다.
- 배급사 관점의 상업 배급 전략과 더 잘 맞는다.
- 시각화와 발표 메시지를 깔끔하게 만들 수 있다.
- 한 학기 프로젝트 규모에 맞게 관리 가능하다.

## 기간 범위 결정
- 전체 분석 기간은 2016~2025로 잡는다.
- 비교 구간은 아래처럼 나눈다.
  - 2016~2019: 코로나 이전
  - 2020~2022: 코로나기
  - 2023~2025: 최근 구간

### 선택 이유
- 10년 범위라 프로젝트 규모가 충분하다.
- 최신 완결 연도인 2025까지 포함한다.
- 최근 산업 변화와 회복 정체를 시기 비교로 설명하기 좋다.

## 데이터 소스 전략
- 1차는 KOBIS만 사용한다.
- 실제 분석 후 부족한 정보가 확인될 때만 보조 소스를 검토한다.

### 선택 이유
- 현재 목적 5개를 설명하는 데 필요한 핵심 변수 대부분을 KOBIS에서 확보할 수 있다.
- 출처가 단순해져 전처리와 발표가 안정적이다.
- 1차 과제에서 데이터 병합 비용보다 EDA 완성도에 집중할 수 있다.

## 데이터 처리 결과
- 연도별 박스오피스 원본은 해당 연도 매출 발생 영화 전체가 포함되어 있다.
- 따라서 재개봉작과 구작이 함께 섞여 있으므로, 최종 분석용은 개봉일 기준으로 다시 필터링했다.

### 처리 결과 요약
- 박스오피스 집계 범위 통합 영화 수: `9815`
- 분석 범위(2016~2025 개봉작) 영화 수: `5176`
- 메타데이터 매칭 성공: `5171`
- 미매칭: `5`

### 최종 분석용 파일
- `data/processed/kobis_master_2016_2025.csv`
- `data/processed/kobis_processing_summary_2016_2025.json`
- `data/processed/kobis_unmatched_movies_2016_2025.csv`

## 해석 시 주의점
- 연도별 박스오피스는 해당 연도 개봉작만 의미하지 않는다.
- 장르는 다중 값(`|`)이 포함될 수 있다.
- 일부 영화는 감독 값이 비어 있다.
- 미매칭 5개는 공연 실황·팬 콘서트 성격의 항목이어서 일반 영화 메타정보와 다를 가능성이 있다.
