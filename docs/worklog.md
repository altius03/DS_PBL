# Worklog

## 운영 원칙
- append-first 원칙으로 기록한다.
- 사실, 제안, 보류를 구분한다.
- 숫자, 날짜, 파일명, 출처는 가능한 구체적으로 남긴다.
- 발표용 문장과 작업 메모는 분리해서 기록한다.
- 대화에서 학습한 개념과 질문-답변 요약은 `conversation-learning-log.md`에 별도로 누적한다.

## 오픈 이슈
- 발표 시간 확정 필요
- 교수님 선호 발표 스타일 확인 필요
- 국내 영화 범위를 어디까지 포함할지 확정 필요
- 사용 데이터 소스에 API 키가 필요한지 확인 필요
- 팀원 역할 분담 확정 필요

## 위험 요소
- 데이터 수집이 늦어지면 PPT 품질이 크게 떨어질 수 있다.
- 해외/국내 데이터를 섞으면 매칭 난도가 올라갈 수 있다.
- 목적을 너무 많이 잡으면 발표가 산만해질 수 있다.
- 결측치가 큰 변수를 핵심 피처로 잡으면 2차 분석이 흔들릴 수 있다.

## 대응 전략
- 1차는 EDA 중심으로 범위를 좁힌다.
- 핵심 변수 위주로 먼저 수집한다.
- 2차는 피처 선택이 가능한 수준의 설명 가능한 분석을 우선한다.
