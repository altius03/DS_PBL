# Analysis Plan

## 1차 과제 목표
- 데이터 구조와 품질을 이해한다.
- 2차 과제에서 바로 사용할 수 있는 핵심 변수들을 확인한다.
- 발표에 넣을 수 있는 시각화 3~5개를 만든다.

## 현재 분석 범위
- 한국 일반영화 중심으로 진행한다.
- 독립영화·예술영화와 외국 영화는 우선 제외한다.
- 범위를 지나치게 넓히지 않고, 배급사 관점에서 비교 가능한 집단을 우선 확보한다.
- 기간은 2016~2025의 10년으로 본다.
- 시기 구간은 2016~2019, 2020~2022, 2023~2025로 나누어 비교한다.
- 데이터 수집은 KOBIS만으로 먼저 진행한다.
- 실제 분석에 필요한 정보가 부족한 경우에만 보조 데이터를 검토한다.

## 1차 과제에서 사용할 목적
1. 장르와 흥행 성과의 관계
2. 개봉 시기와 흥행 성과의 관계
3. 관람등급과 흥행 성과의 관계
4. 스크린 수/상영 횟수와 흥행 성과의 관계
5. 시기 구간별 흥행 패턴 변화

## 목적 활용 방식
- 1차에서는 위 5개를 모두 사용해 EDA를 구성한다.
- 목적들은 병렬적으로 보되, 모두 `배급사 입장에서 흥행 성과를 설명하는 요인 탐색`이라는 하나의 큰 질문 아래 묶는다.
- 시기 구간 비교는 다른 목적들을 보강하는 공통 비교 축으로도 사용한다.
- 2차에서는 위 목적들 중 하나를 중심으로 더 깊게 확장할 수 있다.

## 1차 과제 퀄리티 목표
- 기본 방향은 `완성도 강화형`으로 잡는다.
- 완성도 강화형의 기준:
  - 단순 통계 나열이 아니라, 데이터 구조를 설명하는 스토리라인이 있어야 한다.
  - 그래프 수는 최소 5개, 가능하면 6~7개까지 준비한다.
  - 그래프마다 핵심 해석 문장을 붙인다.
  - 결측치, 분포, 범주형 비교, 관계 시각화가 모두 포함되도록 구성한다.
  - 2차 과제로 이어질 종속변수 후보와 피처 후보를 미리 확인한다.
- 단, 아래는 1차 범위 밖으로 본다.
  - 복잡한 머신러닝 모델 구축
  - 과도한 외부 데이터 결합
  - 설명보다 성능 숫자에 치우친 분석

## 1차 EDA 체크리스트
- 데이터 shape 확인
- 컬럼 타입 정리
- 결측치 개수와 비율 확인
- 중복 여부 확인
- 주요 수치형 변수 기술통계 확인
- 범주형 변수 빈도 확인
- 이상치와 치우친 분포 확인

## 수집 전 최소 컬럼 기준
### 필수 컬럼
- `movie_id`
- `movie_name`
- `open_date`
- `genre`
- `rating`
- `sales_acc`
- `audience_acc`
- `screen_count`
- `show_count`
- `nation`
- `movie_category`

### 보조 컬럼
- `distributor`
- `production_company`
- `director`
- `runtime`
- `year`
- `month`

### 수집 후 파생 컬럼
- `open_year`
- `open_month`
- `quarter`
- `season`
- `covid_period`
- `screen_efficiency`
- `show_efficiency`

## 최소 컬럼 기준을 이렇게 잡는 이유
- 장르, 개봉 시기, 관람등급, 스크린 수/상영 횟수, 시기별 변화라는 목적 5개를 모두 커버해야 한다.
- KOBIS만으로 우선 수집할 수 있는 범위 안에서 필수 변수와 보조 변수를 분리해 관리하는 편이 효율적이다.
- 파생 컬럼은 원본 확보 후 생성하는 편이 재현성과 설명력이 좋다.

## 1차 권장 시각화
- 관객 수 분포 히스토그램
- 매출액 분포 히스토그램
- 장르별 영화 수 막대그래프
- 장르별 평균 또는 중앙 관객 수 비교
- 개봉 월별 영화 수 또는 평균 성과 그래프
- 관람등급별 성과 비교 그래프
- 스크린 수 vs 관객 수 산점도
- 상영 횟수 vs 매출액 산점도

## 2차 과제 기본 방향
- 종속변수 후보를 먼저 정한다.
- 후보 피처를 수치형과 범주형으로 나눈다.
- 피처별로 종속변수와의 관계를 시각화하고 수치로 확인한다.
- 근거가 분명한 피처 2개 이상을 최종 선택한다.

## 2차 종속변수 후보
- `audience_acc`
- `sales_acc`
- `log_audience_acc`
- `log_sales_acc`

## 2차 피처 후보
- 수치형
  - `screen_count`
  - `show_count`
  - `runtime`
  - `month`
- 범주형
  - `genre`
  - `rating`
  - `season`
  - `distributor`
- 파생 피처
  - `screen_efficiency`
  - `show_efficiency`
  - `genre_main`

## 2차 분석 방법
### 분포도
- 종속변수 분포 확인
- 로그 변환 필요성 확인
- 피처별 분포와 이상치 확인

### 상관계수
- 수치형 피처와 종속변수의 상관계수 계산
- 절대값 크기만 보지 말고 해석 가능성도 함께 본다.

### 그룹 분할
- 장르별
- 관람등급별
- 개봉 시기별
- 필요하면 흥행 상위/하위 그룹 분할

## 피처 선택 기준
- 결측치가 지나치게 많지 않을 것
- 종속변수와 관계가 설명 가능할 것
- 그래프와 수치가 같은 방향을 가리킬 것
- 발표에서 직관적으로 설명할 수 있을 것
- 추가 데이터 없이도 재현 가능할 것

## 2차 제출 시 반드시 설명할 것
- 어떤 종속변수를 선택했는가
- 어떤 피처 2개 이상을 선택했는가
- 왜 그 피처를 선택했는가
- 어떤 그래프와 상관계수를 근거로 삼았는가
- 추가 데이터나 피처 가공을 썼다면 왜 필요했는가

## 오늘 기준 최소 액션 플랜
1. 데이터 범위를 국내 영화로 확정
2. 원본 데이터 1~2개 확보
3. 전처리 및 EDA 수행
4. 그래프 3~5개 생성
5. PPT 초안 반영

## 완성도 강화형으로 갈 때 추가 확보할 것
1. 그래프별 해석 문장 작성
2. 결측치/이상치 처리 기준 명시
3. 변수별 의미와 한계 간단 정리
4. 2차에서 쓸 종속변수 후보와 피처 후보 메모
5. 발표 슬라이드 흐름에 맞춘 그림 선별
